# File-monitoring

# Описание
Система мониторинга файлов, разработанная на C++ с использованием библиотеки Qt. Программа отслеживает изменения указанных файлов (размер, существование) и уведомляет наблюдателей об изменениях в реальном времени.

# Функциональность
Добавление файлов - отслеживание новых файлов по указанным путям

Удаление файлов - прекращение отслеживания выбранных файлов

Очистка списка - удаление всех файлов из мониторинга

Мониторинг изменений - автоматическая проверка состояния файлов с заданным интервалом

Уведомление об изменениях - оповещение наблюдателей при:
 - Создании/удалении файла
 - Изменении размера файла
 - Обнаружении пустого файла

Валидация путей - проверка корректности имен файлов

# UML-диаграмма классов

# Основные компоненты UML-диаграммы классов
## IObserver (интерфейс наблюдателя)
 - Базовый класс для всех наблюдателей
 - Определяет метод update() для получения уведомлений

## ConsoleObserver (конкретный наблюдатель)
 - Реализует вывод информации в консоль
 - Отображает:
 - - Путь к файлу
 - - Статус существования
 - - Размер файла
 - - Предупреждение о пустом файле

## FileManager (субъект/издатель)
 - Управление файлами: хранение списка отслеживаемых файлов
 - Мониторинг: периодическая проверка изменений
 - Управление подписками: attach/detach наблюдателей
 - Оповещение: отправка сигналов через Qt сигнально-слотовую систему

# Использование
## Подключение файлов
```
FileManager manager;
ConsoleObserver observer;

manager.attach(&observer);  // Подписка наблюдателя

// Добавление файлов для отслеживания
manager.addFile("C:/path/to/file1.txt");
manager.addFile("C:/path/to/file2.txt");

manager.start();  // Запуск мониторинга
```

## Формат вывода в консоль
```
File: C:/path/to/file.txt
status: exist
size: 1024 bytes

File: C:/path/to/deleted.txt
status: NOT exist
```


# Как работает?
## 1. Инициализация
Создаются объекты FileManager и ConsoleObserver

Наблюдатель подписывается на уведомления через attach()

## 2. Добавление файлов
Проверка корректности имени файла (запрещенные символы)

Проверка на дубликаты

Сохранение информации о файле в m_files

## 3. Процесс мониторинга
Первый проход: сохранение начальных размеров файлов и отправка текущего состояния

Бесконечный цикл: с заданным интервалом проверяются все файлы

Сравнение: текущий размер сравнивается с предыдущим

Оповещение: при обнаружении изменений вызывается notify()

## 4. Уведомление
notify() генерирует сигнал fileChanged()

Сигнал автоматически вызывает слот update() у всех подписанных наблюдателей

Наблюдатель выводит информацию об изменениях в консоль

## 5. Архитектурные особенности
Паттерн "Наблюдатель" (Observer pattern) через Qt сигналы/слоты
 - Асинхронность: мониторинг не блокирует главный поток
 - Реактивность: мгновенное уведомление об изменениях

## Пример работы
Пользователь создает файл test.txt размером 0 байт

Система выводит: warning: file is empty!

Пользователь добавляет текст в файл

Система выводит новый размер: size: 1024 bytes
